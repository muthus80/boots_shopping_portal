## User Flows

### Guest Checkout Flow _(happy path)_
1. **Cart**: User clicks 'Proceed to Checkout' → Navigates to Checkout page _(System checks if cart is empty before allowing navigation.)_2. **Checkout**: User enters guest email and shipping address, clicks 'Continue to Payment' → Frontend calls POST /api/v1/checkout/payment-intent _(Backend creates Stripe PaymentIntent and returns client_secret. UI advances to Stripe Elements step.)_3. **Checkout**: User enters card details in Stripe Elements and clicks 'Pay Now' → Frontend confirms card payment directly with Stripe _(Spinner displays while processing.)_4. **Checkout**: Stripe confirms payment success → Frontend calls POST /api/v1/checkout/confirm with payment_intent_id _(Backend verifies, creates order, and sends email.)_5. **Checkout**: Backend returns 201 Order Created → User sees Order Confirmation success view _(Cart state is cleared in the frontend.)_
**Error scenarios:**
- Card declined by Stripe (show error Alert, keep user on payment step)
- Network error during confirm (show error Alert, allow retry)

### Product Discovery & Cart Flow _(happy path)_
1. **Home**: User types 'hiking' in search bar and presses Enter → Navigates to /products?q=hiking _(SiteHeader triggers navigation.)_2. **ProductList**: User clicks on a specific hiking boot card → Navigates to /products/:id _(Product detail page loads with full data.)_3. **ProductDetail**: User selects size '10', color 'Brown', and clicks 'Add to Cart' → Frontend calls POST /api/v1/cart/items _(Button shows loading state.)_4. **ProductDetail**: API returns 200 with updated cart → Cart icon badge updates, success toast/alert appears _(User remains on the product page.)_
**Error scenarios:**
- Variant out of stock (disable Add to Cart button or show error)
- API error on Add to Cart (show error Alert)
