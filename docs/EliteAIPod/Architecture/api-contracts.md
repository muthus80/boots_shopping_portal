### POST `/api/v1/auth/register`

Validates email uniqueness, hashes password with bcrypt, persists user, sends confirmation email.

**Status codes:** `201` User created successfully, `409` Email already registered, `422` Validation error (password complexity)_Related stories: US-001_
### POST `/api/v1/auth/login`

Validates credentials against bcrypt hash, issues short-lived access JWT and long-lived refresh token.

**Status codes:** `200` Login successful, `401` Invalid credentials, `429` Rate limit exceeded_Related stories: US-002_
### POST `/api/v1/auth/refresh`

Validates refresh token, checks revocation status, issues new access token.

**Status codes:** `200` New access token issued, `401` Refresh token invalid or expired_Related stories: US-013_
### POST `/api/v1/auth/logout`

Marks the refresh token as revoked in the database, effectively ending the session.

**Status codes:** `200` Logged out successfully, `401` Not authenticated_Related stories: US-013_
### GET `/api/v1/categories`

Returns all active product categories with id, name, and slug for navigation.

**Status codes:** `200` Categories returned_Related stories: US-006_
### GET `/api/v1/products`

Returns paginated product cards. Supports category slug, size, color, min_price, max_price filters applied conjunctively.

**Status codes:** `200` Products returned, `404` Category not found_Related stories: US-005, US-006_
### GET `/api/v1/products/search`

Searches product name, brand, and description using PostgreSQL full-text search. Returns matching products or empty array with no_results flag.

**Status codes:** `200` Search results returned (may be empty)_Related stories: US-004_
### GET `/api/v1/products/{product_id}`

Returns complete product data: all images, description, material, features, sizing info, and all available variants (size/color with inventory count).

**Status codes:** `200` Product detail returned, `404` Product not found_Related stories: US-007_
### GET `/api/v1/products/{product_id}/reviews`

Returns paginated list of reviews with average star rating for the product.

**Status codes:** `200` Reviews returned_Related stories: US-008_
### POST `/api/v1/products/{product_id}/reviews`

Creates a review. Verifies that the authenticated user has purchased the product via order_id join. Returns 403 if no matching purchase found.

**Required role:** member**Status codes:** `201` Review created, `401` Not authenticated, `403` User has not purchased this product, `422` Validation error_Related stories: US-008_
### POST `/api/v1/cart/items`

Adds an item to the cart. Creates a new cart if none exists for the session or user. Validates variant availability. Supports both guest (session_id) and authenticated (JWT) carts.

**Status codes:** `200` Item added, updated cart returned, `400` Variant out of stock or invalid quantity, `404` Product variant not found_Related stories: US-009_
### GET `/api/v1/cart`

Returns all items in the current cart (guest session or authenticated user), with per-item details and subtotal.

**Status codes:** `200` Cart returned (empty cart returns items: [])_Related stories: US-010_
### PATCH `/api/v1/cart/items/{item_id}`

Updates the quantity of a specific cart item. Returns updated cart.

**Status codes:** `200` Quantity updated, `400` Invalid quantity, `404` Cart item not found_Related stories: US-010_
### DELETE `/api/v1/cart/items/{item_id}`

Removes a specific item from the cart. Returns updated cart.

**Status codes:** `200` Item removed, `404` Cart item not found_Related stories: US-010_
### POST `/api/v1/checkout/payment-intent`

Fetches cart total, creates a Stripe PaymentIntent server-side. Returns client_secret for Stripe Elements. No card data touches application servers.

**Status codes:** `200` PaymentIntent created, `400` Cart is empty or invalid, `502` Stripe API error_Related stories: US-011_
### POST `/api/v1/checkout/confirm`

Verifies Stripe payment status, creates order record, clears cart, triggers order confirmation email. Handles both guest and authenticated checkout.

**Status codes:** `201` Order created successfully, `400` Payment not succeeded or cart empty, `502` Stripe verification error_Related stories: US-011_
### GET `/api/v1/account/orders`

Returns paginated list of past orders for the authenticated user. Returns empty array with message if no orders placed.

**Required role:** member**Status codes:** `200` Orders returned, `401` Not authenticated_Related stories: US-003_
