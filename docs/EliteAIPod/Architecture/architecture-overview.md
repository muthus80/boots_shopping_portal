## Architecture Pattern

**Pattern:** modular_monolith

A modular monolith is the optimal pattern for this greenfield B2C boots e-commerce application. The 6-week sprint plan covers 15 user stories across five functional domains (auth, products, cart, checkout, reviews). A modular monolith delivers all inter-module communication in-process (zero network latency between cart and product lookups), drastically simplifies the Docker Compose deployment topology, and reduces operational complexity for the initial launch. Each domain (auth, products, cart, checkout, reviews, account) is encapsulated as a Python package with its own routes, services, and repositories — providing clear module boundaries that allow future extraction into microservices if scaling demands it. FastAPI's async support and SQLAlchemy 2.x provide the performance headroom needed to satisfy the LCP 2.5s NFR without pre-mature infrastructure complexity. The single PostgreSQL instance simplifies ACID transactions across cart-to-order conversion and purchase-verification for reviews.

## Infrastructure

- **Deployment:** Docker Compose orchestrating four containers: Nginx (reverse proxy, ports 80/443), FastAPI backend (Uvicorn, port 8000), React SPA (Nginx static serving, port 3000), PostgreSQL (port 5432). Single docker-compose.yml with named volumes for PostgreSQL data persistence. Environment variables via .env file for secrets. Alembic migrations run as an init container before FastAPI starts.
- **Scaling:** Horizontal scaling of FastAPI via multiple Uvicorn workers behind Nginx upstream. PostgreSQL read replicas for catalog queries under load. CDN (e.g., Cloudflare) for static assets and product images to satisfy LCP 2.5s NFR. Connection pooling via SQLAlchemy pool_size and max_overflow.
- **Caching:** none

## Security Considerations

- Passwords hashed with bcrypt (cost factor 12) before storage — never stored in plaintext
- JWT access tokens expire in 30 minutes; refresh tokens hashed before DB storage and revocable
- Stripe Elements used for card capture — no card data ever touches application servers (PCI-DSS SAQ A compliant)
- All API endpoints served over HTTPS via Nginx TLS termination
- Rate limiting on auth endpoints (5 req/min for login, 10/min for register) to prevent brute force
- SQL injection prevention via SQLAlchemy parameterized queries — no raw string interpolation
- CORS configured to allow only the application's frontend origin
- Secrets (DB password, JWT secret, Stripe keys, email API key) stored in environment variables, never in code
- WCAG 2.1 AA accessibility requirements satisfied via ARIA labels, keyboard navigation, and focus management
